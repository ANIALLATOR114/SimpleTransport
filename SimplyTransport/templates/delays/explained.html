Historical Delay Data

Every single minute, SimplyTransport is collecting and aggregating delay data from the Realtime API. This data is stored and then used to calculate historical statistics over time.

This data is inserted into a database with millions of entries, which is is growing all the time with thousands, or tens of thousands, of new data points every hour.

You can see the number of samples being recorded in the Events page which records each update under timeseries.delays.recorded.

Definitions

Unit of Measurement Unless otherwise specified, all values are recorded in seconds.

Average The average delay across all samples.

50th Percentile The delay value below which 50% of the samples fall.

75th Percentile The delay value below which 75% of the samples fall.

90th Percentile The delay value below which 90% of the samples fall.

Standard Deviation A measure of the amount of variation or dispersion of a set of values.

Minimum The single minimum delay value recorded.

Maximum The single maximum delay value recorded.

Samples The total number of samples recorded.

TimeStamp The exact time at which the data was recorded.

Delay in Seconds The negative or positive delay value in seconds.

Limitations

The data is collected from the Realtime API, which is subject to the accuracy of the data provided by the transport operators (TFI).

Not every vehicle is equipped with a GPS tracker, so some vehicles may not be represented in the data or may have only partial data available.
